functools 模块

functools 的实现主要依赖于 Python 的函数式编程特性（如闭包、装饰器、一等函数）以及一些运行时的元编程技术（如反射、动态绑定）。

functools 的核心实现技术包括：

  1. 闭包与装饰器：如 lru_cache、partial、wraps。
  2. 元编程：动态生成方法（total_ordering）或修改类属性（singledispatch）。
  3. 数据结构应用：使用 OrderedDict 实现 LRU 缓存。
  4. 函数式编程：通过高阶函数增强函数复用性（如 partial、reduce）。

functools 模块应用于高阶函数，即参数或（和）返回值为其他函数的函数。 通常来说，此模块的功能适用于所有可调用对象。

lru_cache 与 cache

  • **lru_cache**：基于 LRU（最近最少使用）算法缓存函数结果，避免重复计算，适用于递归函数或高计算成本的函数。
  • **cache**（Python 3.9+）：lru_cache 的简化版，默认无限缓存（maxsize=None）。

主要原理

  • LRU 缓存机制：使用 collections.OrderedDict 维护缓存项，最近访问的项移至链表尾部，超出容量时删除头部（最久未使用项）。
  • 参数哈希：通过 _make_key 函数将参数（包括位置参数和关键字参数）转换为可哈希的键，确保不同参数组合对应唯一缓存项。

singledispatch 的动态分派机制

使用 @singledispatch 装饰的普通函数会变成泛函数（generic function）：实现函数或方法的单分派泛函数（根据第一个参数的类型选择不同的实现）

  1. @singledispatch 标记处理 object 类型的基函数。
  2. 各个专门函数使用 @«base_function».register(«type») 装饰。

singledispatch 机制的一个显著特征是，你可以在系统的任何地方和任何模块中注册专 门函数。如果后来在新的模块中定义了新的类型，可以轻松地添加一个新的专门函数来处 理那个类型。此外，你还可以为不是自己编写的或者不能修改的类添加自定义函数。

装饰器元信息保留：wraps

在装饰器中保留被装饰函数的元信息（如 __name__、__doc__），避免调试时信息丢失。

参数固化：partial

固定函数的部分参数，生成新的可调用对象，简化函数调用（如为回调函数预设参数）。

累积计算：reduce

对可迭代对象的元素进行累积计算（类似数学中的 “累加”“累乘”），Python 3 中需从 functools 导入。

自动生成比较方法：total_ordering

简化类的比较方法定义，只需实现 __eq__ 和一个其他比较方法（如 __lt__），自动生成剩余方法（__le__、__gt__ 等）。
