变量的作用域——LEGB 规则

在 Python 中，变量作用域（Variable Scope） 指的是变量的有效范围，即程序中可以访问该变量的区域。理解作用域有助于避免变量命名冲突、合理管理内存，并写出更健壮的代码。Python 使用 LEGB 规则 来查找变量，即按照 Local（局部）、Enclosing（嵌套函数/闭包）、Global（全局）、
Built-in（内置）的顺序逐层搜索， Local → Enclosing → Global → Built-in。

一、LEGB 规则详解

1. L：Local（局部作用域）

  • 范围：函数内部定义的变量（包括参数），仅在函数体内有效。

3. G：Global（全局作用域）

  • 范围：模块（.py 文件）顶层定义的变量，在整个模块内有效，但函数内部不能直接修改（需用 global 声明）。

二、变量作用域的分类

1. 局部变量（Local Variables）

  • 定义位置：函数内部（包括参数）。
  • 生命周期：随函数调用创建，结束后销毁。
  • 特点：
    • 仅在函数内可见。
    • 不同函数的同名局部变量相互独立。

2. 全局变量（Global Variables）

  • 定义位置：模块顶层（函数外部）。
  • 访问规则：
    • 函数内部可直接读取全局变量，但不能直接修改（除非用 global 声明）。
    • 示例：

3. 非局部变量（Nonlocal Variables）

  • 定义位置：嵌套函数的外层函数中（非全局）。
  • 修改规则：内层函数若要修改外层函数的变量，需用 nonlocal 声明。
  • 示例：

三、作用域相关的关键字

1. global

  • 作用：在函数内部声明某个变量为全局变量，允许修改全局作用域中的变量。
  • 注意：
    • 若函数内仅读取全局变量，无需 global；若要修改，则必须声明。
    • 不能在嵌套函数中使用 global 声明外层函数的变量（需用 nonlocal）。

2. nonlocal

  • 作用：在嵌套函数的内层函数中，声明变量来自外层函数（非全局），允许修改外层函数的变量。
  • 限制：
    • 不能用于修改全局变量。
    • 外层函数必须存在该变量，否则报错。
